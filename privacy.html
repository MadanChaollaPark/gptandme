Privacy Policy

GPTandME

Last updated: 2026-07-10

Local data handling

This extension handles usage data locally in your browser so it can count prompts and show usage statistics. It does not transmit, sell, or share user data. It does not send data to any server and does not use analytics, advertising, or tracking services.

Data storage

All usage data is stored locally on your device using the Chrome Storage API. This includes prompt counts, daily/weekly/monthly totals, per-service counts for ChatGPT, Claude, Gemini, Perplexity, and Grok, model-count summaries when available, hourly/session statistics, non-sensitive diagnostics such as the last counted time/source and last supported host, and local preferences such as the in-page counter setting. To prevent transport retries from being counted twice, GPTandME also keeps up to 500 opaque event identifiers locally with a 24-hour deduplication window; expired identifiers are pruned during hourly maintenance, startup, or the next write, and reset controls clear them. No data leaves your browser unless you manually export a CSV file from the popup.

Prompt content

On supported chat services, the extension may inspect outgoing request data locally to distinguish a new user send from an automatic retry and to read an opaque request identifier or model label when available. Only non-content metadata is emitted to the extension. The extension does not store prompt text or a page's full URL path, and it does not send prompt text anywhere.

Extension surfaces

Version 1.4.0 exposes local usage information through the extension badge, the extension popup, and an optional in-page widget/counter on supported chat hosts. The popup shows today's and all-time prompt counts for ChatGPT, Claude, Gemini, Perplexity, and Grok, and may show a local OpenAI API cost proxy based on prompt-count assumptions; it is not actual billing data. The widget renders inside a closed browser shadow root so the host page cannot read its count. The widget is optional; the badge and popup remain the primary supported surfaces.

CSV backups

Manual CSV exports use the columns date, provider, model, and count. Legacy GPTandME CSV files without a provider column remain importable and are assigned the provider value "unknown" because the extension cannot safely infer historical service attribution.

Supported hosts

The extension only runs on supported AI chat hosts: chatgpt.com, chat.openai.com, claude.ai, gemini.google.com, perplexity.ai, www.perplexity.ai, and grok.com. It does not run on OpenAI platform, API, billing, help, documentation, or unrelated web pages.

Permissions

  • storage — used to save your prompt counts locally.
  • webRequest — used locally for ChatGPT request detection so user-sent prompts can be counted more reliably.
  • activeTab — used only when you open the popup so it can label whether the current tab is a supported host.
  • alarms — used locally to refresh the badge count after date changes while the browser is running.
  • host_permissions — limited to chatgpt.com, chat.openai.com, claude.ai, gemini.google.com, perplexity.ai, www.perplexity.ai, and grok.com so the content script can detect user sends on supported chat sites.

Third-party services

This extension does not integrate with any third-party analytics, advertising, or tracking services. Version 1.4.0 adds only the narrow grok.com host permission required for Grok counting and does not add external usage telemetry.

Limited use

The use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

User control

You can reset your counts at any time using the "Reset today" or "Reset all" buttons in the popup. You can also export or import a local CSV manually. Uninstalling the extension removes the locally stored extension data from the browser profile.
